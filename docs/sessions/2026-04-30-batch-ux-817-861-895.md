## #817 `/macro` — Command Recording & Playback

**File:** `Macros.fs` · **What it does:** Record sequences of user commands into named macros, then replay them verbatim.

### Usage

```
/macro record myquery          # Start recording (stops at next /macro stop)
<run commands...>
/macro stop                    # Save sequence to named macro
/macro play myquery            # Replay: injects all commands back via ReadLine queue
/macro list                    # Show all recorded macros
/macro remove myquery          # Delete macro
```

### Implementation

- **State:** `Dictionary<string, string list>` (macro name → command list)
- **Recording:** LineReader collects input during `recordingMode` flag
- **Playback:** Commands queued via `inputQueue` injected before ReadLine, executed atomically
- **Scope:** Session-local; no persistence

### Known gaps

- **#896** (follow-up): no unit tests for record/play/list/remove workflows
- No syntax for: macro parameters, conditional playback, nested macros
- No deletion confirmation (destructive)

## #861 `/theme bubbles` — AI Response Panels

**File:** `Render.fs` · **What it does:** Wrap AI responses in styled panels (rounded border, dim header with " Claude " label).

### Usage

```
/theme bubbles                 # Toggle bubblesMode on (default off)
<AI responds>                  # Rendered in bordered Panel(rounded, dim)
```

### Implementation

- **Toggle:** `/theme bubbles` sets `session.bubblesMode ← !bubblesMode`
- **Render:** `Panel(rounded=true, dim=true)` with `" Claude "` header wraps `content`
- **Styling:** Spectre.Console theming; inherits session color palette
- **Scope:** Session-local; resets on `/new`

### Known gaps

- **#897** (follow-up): no persistence to config/session state file
- No per-response toggle (all-or-nothing per session)
- Bubble header text hardcoded (not customizable)

## #895 `/watch debounce` — File Monitor Rate-Limit

**File:** `FileWatcher.fs` · **What it does:** Suppress duplicate file change notifications on macOS (300ms debounce window).

### Problem solved

macOS `FileSystemWatcher` fires 2–5 events per single file save; debounce reduces noise when tools trigger watches during edit/save cycles.

### Implementation

- **Mechanism:** Per-file `lastFire` timestamp tracking
- **Window:** 300ms; events within window discarded
- **Scope:** Applied to all watched paths (Read, Edit, Glob tool outputs)

### Known gaps

- Debounce interval hardcoded; no user config
- No telemetry (cannot verify actual event reduction in production)
