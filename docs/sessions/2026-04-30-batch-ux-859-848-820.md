## Features Shipped

### 1. Emoji ASCII Fallback (#859)

Terminal emoji (✅, ❌, 🔧, etc.) are now replaced with ASCII equivalents ([ok], [x], [cfg]) when the terminal doesn't support UTF-8.

**How it works:**
- Auto-detection via `TERM_PROGRAM` (iTerm, WezTerm, vscode), `COLORTERM=truecolor`, or `LANG`/`LC_ALL` UTF-8 suffix.
- 20-entry lookup table in `EmojiMap.fs` (no generics; AOT-safe).
- Applied at render time in `MarkdownRender.fs` via `emojiNormalise`.

**Usage:** Automatic; no user action needed.

**Known issues:**
- #893: Emoji config not yet user-toggleable; detection is fixed per session.
- Additional emoji pairs (25+) blocked until emoji config gains a `/set emoji` command.

### 2. Nocturne Theme (#848)

New dark theme with pale gold headings (#b8a060) and mid-blue code blocks (#4a90d9).

**How it works:**
- `initTheme(cfg.Ui.Theme)` wires config value into `MarkdownRender.activeTheme`.
- Heading colors (H1: bold underline, H2: bold, H3+: dim) applied when `activeTheme = "nocturne"`.
- Code block color switches from teal (default) to mid-blue.

**Usage:** Set in `~/.fugue/config.json`:
```json
"ui": { "theme": "nocturne" }
```

**Known issues:**
- #894: Full Nocturne palette (links, inline code, tables) not yet colored.
- Only headings and code blocks styled; fallback is default theme.

### 3. `/watch` File Trigger (#820)

Watch a file for changes and auto-run a command on save (polling via `FileSystemWatcher.LastWrite`).

**Commands:**
- `/watch <path> <cmd>` — Start watching; re-run `<cmd>` on file save.
- `/unwatch <path>` — Stop watching.
- `/watches` — List active watches.

**How it works:**
- `FileWatcher.fs` maintains a channel of pending triggers.
- REPL drains triggers before each ReadLine (non-blocking).
- Drained commands are auto-executed with `[watch] <cmd>` label.
- `/new` session reset clears all watches.

**Example:**
```
/watch src/main.fs "!cargo build"
# → Each save triggers: !cargo build
```

**Known issues:**
- #895: No debounce; rapid saves may queue multiple triggers.
- `LastWrite` filter only; renames/deletes not watched.
- Watch survives across REPL loops but is cleared on `/new`.
